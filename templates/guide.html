🚀 1. เริ่มต้นใช้งาน

ติดตั้งบนเครื่อง

  1. Clone โปรเจค
    git clone https://github.com/dmz2001TH/facebook-ad-scaler.git
    cd facebook-ad-scaler
  2. ติดตั้ง dependencies
    pip install -r requirements.txt
  3. คัดลอกไฟล์ config
    cp .env.example .env
  4. แก้ไข .env ตามต้องการ (ดูหัวข้อ "ตั้งค่าระบบ")
  5. รัน server
    python app.py

เปิด browser ไปที่ http://localhost:8080

ติดตั้งด้วย Docker

docker build -t ad-scaler .
docker run -p 8080:8080 -v ad-scaler-data:/data ad-scaler

ติดตั้งแบบ Production

pip install -r requirements.txt gunicorn
./start.sh
start.sh จะตรวจว่ามี gunicorn หรือไม่ ถ้ามีจะใช้ gunicorn อัตโนมัติ ถ้าไม่มีจะ fallback ไปใช้ Flask dev server

📘 2. ตั้งค่า Facebook API

ขั้นตอนที่ 1: สร้าง Facebook App

  1. ไปที่ developers.facebook.com
  2. กด Create App → เลือก Business → ตั้งชื่อ app
  3. ไปที่ Settings > Basic → จด App ID และ App Secret

ขั้นตอนที่ 2: ขอ Permission

  1. ไปที่ App Review > Permissions and Features
  2. ขอ permission: ads_management, ads_read, business_management
  3. เพิ่ม Marketing API ใน Products

ขั้นตอนที่ 3: สร้าง Access Token

  1. ไปที่ Graph API Explorer
  2. เลือก app ของคุณ
  3. เลือก permission: ads_management, ads_read
  4. กด Generate Access Token
Token แบบนี้อายุสั้น (~1-2 ชม.) สำหรับใช้งานจริง ควรใช้ Long-lived token หรือ System User token

ขั้นตอนที่ 4: ใส่ Token ในระบบ

ไปที่ Settings → วาง Facebook Access Token → กด Save

📢 4. จัดการ Campaigns

ไปที่เมนู Campaigns จะเห็นรายการ campaigns ทั้งหมด

เปิด / ปิด Campaign

เลือก campaign → กดปุ่ม Enable หรือ Pause ระบบจะบันทึกใน audit trail อัตโนมัติ

เปลี่ยนงบประมาณ

เลือก campaign → แก้ไข Daily Budget หรือ Lifetime Budget → กด Save

ทุกการเปลี่ยนแปลงงบจะถูกบันทึกใน Budget Calendar อัตโนมัติ ทำให้เห็นผลลัพธ์หลังเปลี่ยนงบ

🧠 5. Smart Scaling

ระบบวิเคราะห์และแนะนำการ scale อัตโนมัติ — ดู ROAS trend แล้วตัดสินใจแทนคุณ

ตั้งค่า

Parameterคำอธิบายค่าแนะนำ
max_budget_increase_pctเพิ่มงบได้สูงสุดกี่ %20-30%
min_roas_thresholdROAS ขั้นต่ำที่จะ scale2.0x
lookback_daysดูข้อมูลย้อนหลังกี่วัน7 วัน
kill_loss_thresholdขาดทุนเกินเท่าไหร่ให้ปิด (บาท)500 บาท

วิธีใช้

  1. กด Analyze → ระบบจะวิเคราะห์ campaigns ทั้งหมด
  2. ดูคำแนะนำ:
    • Increase — ROAS ดี แนะนำเพิ่มงบ 20-30%
    • Decrease — ROAS ลดลง แนะนำลดงบ
    • Kill — ขาดทุนเกิน threshold แนะนำปิด
  3. กด Execute เพื่อทำจริง

Kill Switch — ปิดฉุกเฉิน

ถ้าทุกอย่างพัง กด Kill Switch → ระบบจะ pause ทุก campaign ที่กำลังขาดทุนทันที

Kill Switch จะหยุดเฉพาะ campaigns ที่ขาดทุนเกิน threshold ไม่ใช่ปิดหมด — ตัวที่กำไรยังวิ่งต่อ

⚙️ 6. Auto-Rules

ตั้งกฎให้ระบบทำงานอัตโนมัติ — ไม่ต้องนั่งเฝ้าจอ

ตัวอย่างกฎที่แนะนำ

กฎเงื่อนไขการกระทำความถี่
ปิด Ad CPC สูงCPC > ฿20Pause Campaignทุก 30 นาที
เพิ่มงบ ROAS ดีROAS > 3.0xเพิ่มงบ 20%ทุก 3 ชม.
แจ้งเตือน CTR ตกCTR < 0.5%ส่งแจ้งเตือนทุก 1 ชม.
ปิด Ad Spend เกินSpend > ฿1000 & ROAS < 1.5Pause + แจ้งเตือนทุก 1 ชม.

รูปแบบเงื่อนไข (Condition)

{"metric": "cpc", "operator": ">", "value": 20}

Metrics ที่ใช้ได้: cpc, ctr, cpm, roas, spend, impressions

Operators: > < >= <= == !=

รูปแบบการกระทำ (Action)

// ปิด campaign
{"type": "pause_campaign"}

// เพิ่มงบ 20%
{"type": "adjust_budget", "change_pct": 20}

// ลดงบ 30%
{"type": "adjust_budget", "change_pct": -30}

// แจ้งเตือน
{"type": "notify", "message": "CPC สูงเกินไป!"}

รัน Rule ด้วยมือ

ไปที่ Rule ที่ต้องการ → กด Run Now

Export / Import Rules

สามารถ export rules เป็น JSON แล้ว import กลับได้ — เหมาะสำหรับย้ายเครื่องหรือแชร์ให้ทีม

💬 7. AdsGPT — AI ช่วยวิเคราะห์

AI ผู้ช่วยที่ตอบคำถามเรื่อง Facebook Ads ได้ทันที

ตัวอย่างคำถามที่ถามได้

คำถามระบบจะตอบ
"แนะนำวิธี scale ads"4 ขั้นตอนการ scale + ข้อควรระวัง
"CPC เท่าไหร่ถึงดี?"เกณฑ์ CPC/CTR/CPM/ROAS สำหรับตลาดไทย
"ตั้ง auto-rule ยังไง?"ตัวอย่างกฎ 4 แบบที่แนะนำ
"ad ตัวไหนควรปิด?"วิเคราะห์จาก metrics (ถ้าต่อ OpenAI)

ต่อ OpenAI (ตัวเลือก)

ถ้าต้องการให้ AI ตอบได้ฉลาดขึ้น:

  1. ไปที่ .env ไฟล์
  2. เพิ่ม: OPENAI_API_KEY=sk-your-key-here
  3. รีสตาร์ท server
ถ้าไม่ต่อ OpenAI ระบบจะใช้ fallback responses ที่เขียนไว้ (ภาษาไทย) — ยังใช้งานได้ แค่ไม่ฉลาดเท่า

🧪 8. A/B Test Tracker

เปรียบเทียบ ad sets 2 ตัว ว่าตัวไหนดีกว่า — ระบบ declare winner ให้อัตโนมัติ

วิธีใช้

  1. ไปที่เมนู Experiments → กด Create
  2. เลือก Ad Set A กับ Ad Set B
  3. เลือก metric ที่จะเปรียบเทียบ (CPC, CTR, ROAS)
  4. ตั้ง duration (กี่วัน)
  5. รอให้ครบ → กด Conclude → ระบบจะดึงข้อมูลจริงจาก Facebook แล้วบอก winner

😴 9. Creative Fatigue Detection

ตรวจจับโฆษณาที่ "ตาย" แล้ว — CTR ลดฮวบใน 3 วัน

ระบบจะบอก

  • Ad ไหน fatigued — CTR ลด > 30% เทียบกับ 7-day average
  • Severity — warning (ลด 30-50%) หรือ critical (ลด > 50%)
  • Recommendation — แนะนำให้ทำอะไร (เช่น "เปลี่ยน creative")
ควรเช็ค ทุก 1-2 วัน — creative fatigue เป็นสาเหตุอันดับ 1 ที่ทำให้ ads แพงขึ้นโดยไม่รู้ตัว

🚨 10. Anomaly Detection

ตรวจจับ metrics ที่ผิดปกติ ด้วยวิธีทางสถิติ (Z-score)

ระบบจะตรวจ

  • CPC ผิดปกติ — สูงหรือต่ำกว่าปกติ > 2 standard deviations
  • CPM ผิดปกติ
  • CTR ผิดปกติ
  • Spend ผิดปกติ

Severity

  • warning — z-score 2.0-2.5 (ผิดปกติเล็กน้อย)
  • critical — z-score > 2.5 (ผิดปกติรุนแรง)

⏱️ 11. Budget Pacing & Calendar

Budget Pacing

ดูว่าวันนี้ใช้งบไปเท่าไหร่แล้ว แต่ละ campaign:

  • onschedule — ใช้ตามแผน
  • underspending — ใช้น้อยกว่าที่ควร
  • overspending — ใช้มากกว่าที่ควร

Budget Calendar

ดูประวัติการเปลี่ยนงบ + ผลลัพธ์ — เปลี่ยนวันไหนแล้ว ROAS/CPC เป็นยังไง

🌐 15. Deploy ขึ้น Server จริง

Docker (แนะนำ)

docker build -t ad-scaler .
docker run -d --name ad-scaler -p 8080:8080 -v ad-scaler-data:/data ad-scaler

Gunicorn (บน VPS)

pip install -r requirements.txt gunicorn
./start.sh

systemd (รันเป็น service)

สร้างไฟล์ /etc/systemd/system/ad-scaler.service:

[Unit]
Description=Facebook Ad Scaler
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/facebook-ad-scaler
EnvironmentFile=/opt/facebook-ad-scaler/.env
ExecStart=/opt/facebook-ad-scaler/start.sh
Restart=always

[Install]
WantedBy=multi-user.target
sudo systemctl enable ad-scaler
sudo systemctl start ad-scaler

Nginx Reverse Proxy

server {
listen 80;
server_name ads.yourdomain.com;
location / {
proxy_pass http://127.0.0.1:8080;
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
}
}

❓ 16. FAQ

ไม่มี Facebook Token ใช้งานได้มั้ย?

ได้บางส่วน — Dashboard, Rules, Settings ใช้ได้ แต่ข้อมูลจริงจาก Facebook จะไม่แสดง

OpenAI Key จำเป็นมั้ย?

ไม่จำเป็น — AdsGPT จะใช้ fallback responses (ภาษาไทย) ถ้าไม่มี key

ข้อมูลเก็บที่ไหน?

SQLite database (scaler.db) — ไฟล์เดียว ไม่ต้อง setup database server

รันพร้อมกันหลาย server ได้มั้ย?

SQLite ไม่รองรับ — ถ้าต้องการหลาย server ต้อง migrate ไป PostgreSQL

Rule รันอัตโนมัติมั้ย?

ใช่ — ถ้าตั้ง schedule (interval_minutes) ไว้ scheduler จะรันให้ทุก N นาที

Backup ยังไง?

copy ไฟล์ scaler.db เก็บไว้:

cp scaler.db scaler_backup_$(date +%Y%m%d).db

Rate Limit reset ตอนไหน?

ทุก 60 วินาที (in-memory, reset ถ้า restart server)

ดู log ที่ไหน?

ไฟล์ scaler.log หรือดูผ่าน terminal ที่รัน server

Facebook Ad Scaler — เครื่องมือจัดการโฆษณา Facebook อัตโนมัติ
